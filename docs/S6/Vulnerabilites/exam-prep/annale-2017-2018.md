# Annale 2017-2018 -- Analyse

> Source : Copie de 2017-2018.pdf
> Note : ce fichier est une analyse basee sur le format PDF de l'annale

---

## Structure de l'annale

L'annale 2017-2018 est l'une des plus anciennes disponibles. Elle permet de comprendre le format general du DS et les types de questions posees.

---

## Types de questions recurrentes depuis 2017

### Partie pratique (style Olivier Heen)

1. **Identification d'attaques** : des lignes de log sont presentees, il faut classifier chaque attaque (XSS, injection SQL, injection de commandes, CSRF, collecte d'informations).

2. **Injection SQL** : on donne une requete PHP/SQL vulnerable et on demande :
   - Que se passe-t-il si on entre `'` ?
   - Comment se connecter en admin ?
   - Proposer une correction

3. **Qualification CVSS** : decrire les metriques pour une vulnerabilite donnee.

### Partie formelle (style Barbara Fila)

1. **Analyse de protocole** : un protocole est donne en notation MSC, et on demande :
   - Le secret de tel message est-il garanti pour tel role ?
   - La weak aliveness est-elle satisfaite ?
   - Proposer une attaque ou prouver que le protocole est correct

2. **Inference de messages** : un ensemble de messages connus est donne, on demande ce qu'on peut en deduire.

3. **Correction de protocole** : modifier un protocole vulnerable pour corriger une faille.

---

## Conseils specifiques pour ce format

1. **Lire toutes les questions avant de commencer** : certaines questions sont liees entre elles
2. **Pour les logs** : etre systematique, classer dans l'ordre
3. **Pour les protocoles** : toujours redessiner le MSC proprement
4. **Justifier** : un point de DS sans justification ne vaut souvent rien
5. **Correction participative** : dans les annales plus recentes, des corrections collaboratives existent -- les consulter pour comprendre les attentes des enseignants

---

## Questions types a preparer

| Question | Competence requise |
|----------|-------------------|
| Classifier un log en XSS/SQL/CMD/CSRF/INFO | Reconnaitre les patterns |
| Ecrire un payload SQL pour bypass login | Comprendre la concatenation |
| Calculer un score CVSS | Connaitre les metriques |
| Verifier le secret d'un nonce | Appliquer les regles d'inference |
| Trouver une attaque MitM | Dessiner les sessions paralleles |
| Corriger un protocole | Identifier l'information manquante |
| Analyser un systeme de mots de passe | Kerckhoffs, espace de recherche |
