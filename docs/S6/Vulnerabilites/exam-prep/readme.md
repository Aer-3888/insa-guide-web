## Structure typique du DS

Le DS est divise en deux parties distinctes :

### Partie Olivier Heen (~10 pts)

| Exercice type | Points | Competence |
|-------------- |--------|-----------|
| Classifier des logs d'attaque | 2-4 | Identifier XSS, SQL, CMD, CSRF, INFO |
| Injection SQL ou commande | 2-4 | Ecrire des payloads, proposer des protections |
| Scenario IoT / CVSS | 2-4 | Qualifier des vulnerabilites, calculer CVSS |
| Securite cloud | 2 | Modeles de menaces, bonnes pratiques |

### Partie Barbara Fila (~10 pts)

| Exercice type | Points | Competence |
|-------------- |--------|-----------|
| Analyse de protocole (MSC) | 4-6 | Verifier secret, authentification |
| Trouver une attaque | 2-4 | Decrire un MitM avec un MSC |
| Proposer une correction | 2 | Modifier le protocole pour le securiser |
| Mots de passe / authentification | 2-4 | Stockage, cracking, proprietes d'auth |

J-7 :  Relire les guides chapitres 1 a 8 (partie Olivier)

J-2 :  Faire les annales en temps limite (2h)
J-1 :  Relire le cheat sheet et preparer ses notes
Jour J: Apporter ses documents organises (cheat sheet imprime)
